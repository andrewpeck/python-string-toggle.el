Toggle Python strings at point between triple-quoted and implicit concatenation forms.

Disclaimer: this package is largely AI written. It was created to solve a very specific problem and did that well.

* Overview

Two representations are supported:

Triple-quoted:
#+begin_src python
"""Pellentesque dapibus suscipit ligula.  Donec posuere augue in quam.  Etiam vel tortor sodales tellus ultricies commodo.  Suspendisse potenti.  Aenean in sem ac leo mollis blandit.  Donec neque quam, dignissim in, mollis nec, sagittis eu, wisi.  Phasellus lacus.  Etiam laoreet quam sed arcu.  Phasellus at dui in ligula mollis ultricies.  Integer placerat tristique nisl.  Praesent augue.  Fusce commodo.  Vestibulum convallis, lorem a tempus semper, dui dui euismod elit, vitae placerat urna tortor vitae lacus.  Nullam libero mauris, consequat quis, varius et, dictum id, arcu.  Mauris mollis tincidunt felis.  Aliquam feugiat tellus ut neque.  Nulla facilisis, risus a rhoncus fermentum, tellus tellus lacinia purus, et dictum nunc justo sit amet elit."""
#+end_src

Implicit concatenation:
#+begin_src python
(
    "Pellentesque dapibus suscipit ligula. Donec posuere augue in quam. Etiam"
    "vel tortor sodales tellus ultricies commodo. Suspendisse potenti. Aenean"
    "in sem ac leo mollis blandit. Donec neque quam, dignissim in, mollis nec,"
    "sagittis eu, wisi. Phasellus lacus. Etiam laoreet quam sed arcu. Phasellus"
    "at dui in ligula mollis ultricies. Integer placerat tristique nisl."
    "Praesent augue. Fusce commodo. Vestibulum convallis, lorem a tempus"
    "semper, dui dui euismod elit, vitae placerat urna tortor vitae lacus."
    "Nullam libero mauris, consequat quis, varius et, dictum id, arcu. Mauris"
    "mollis tincidunt felis. Aliquam feugiat tellus ut neque. Nulla facilisis,"
    "risus a rhoncus fermentum, tellus tellus lacinia purus, et dictum nunc"
    "justo sit amet elit."
)
#+end_src

Place point anywhere inside the string and call ~python-string-toggle~ to switch between them.

Behaviour:
- Lines are stripped of leading whitespace when converting to implicit form.
- No ~\n~ escapes are inserted between lines; blank lines in the source produce a ~"\n"~ fragment.
- A single long line is word-wrapped at ~fill-column~ before conversion.
- String prefixes (~f~, ~r~, ~b~, etc.) are preserved.
- Surrounding parentheses are added or removed automatically.
